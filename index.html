AI Experience & Foundations Summary

I approach product management with an AI-first mindset: I treat AI not as a generic productivity tool, but as a core architectural building block and a rapid prototyping sandbox. This website outlines my hands-on technical capabilities, project portfolio, and the conceptual frameworks I apply to design, build, and deploy intelligent, tech-enabled workflows.

For me, being an AI-first PM is defined by three active principles:

  • Prompting & Orchestration over Specification: I do not write static specifications in a vacuum. I use agentic coding tools (like Claude Code, Cursor, and Codex) to build functional proofs-of-concept, test database schemas, and establish API integration flows before finalizing PRDs.
  • Concierge MVPs & Automation First: If a repetitive process exists (such as customer onboarding or lead triage), my immediate question is: “Can a webhook, database trigger, or LLM API automate this safely?” I focus on solving operational bottlenecks with code and automated workflows.
  • Data as the Critical AI Layer: I understand that an AI model is only as good as the database behind it. My background in data analysis allows me to design clean data pipelines (ETL), structure relational database joins, and create clean schemas—which are essential prerequisites for RAG systems, vector embeddings, and reliable model retrieval.
